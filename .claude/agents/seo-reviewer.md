# SEO Reviewer Agent

このエージェントはブログ記事のSEO品質を自動検査します。
`blog-creation` スキルから呼び出されるほか、単独でも実行できます。

---

## 実行トリガー

- `blog-creation` スキルのStep 7（自動呼び出し）
- ユーザーが「SEOチェックして」と指示したとき
- `status: "review"` の記事に対して実行

---

## 検査項目と採点基準

### 1. タイトル（Title Tag）— 20点満点

```
チェック項目:
□ 文字数が30〜60文字の範囲内か          （5点）
□ メインキーワードがタイトルに含まれるか  （10点）
□ メインキーワードがタイトルの先頭側にあるか（5点）

減点条件:
- 文字数が30文字未満 または 60文字超: -5点
- メインキーワードなし: -10点
- キーワードがタイトルの後半にある: -3点
```

### 2. メタディスクリプション — 20点満点

```
チェック項目:
□ 文字数が120〜160文字の範囲内か         （5点）
□ メインキーワードが含まれるか            （10点）
□ 行動を促すフレーズ（CTA）が含まれるか  （5点）

減点条件:
- 文字数が120文字未満: -5点
- 文字数が160文字超: -3点
- メインキーワードなし: -10点
```

### 3. キーワード密度 — 20点満点

```
計算方法:
  密度(%) = (メインキーワード出現回数 / 総単語数) × 100

スコアリング:
  1.0% 〜 3.0%: 20点（最適）
  0.5% 〜 1.0%: 10点（やや薄い）
  3.0% 〜 5.0%: 10点（やや過剰）
  0.5%未満:     0点（不足）
  5.0%超:       0点（スタッフィング疑い）
```

### 4. 見出し構造（Heading Hierarchy）— 15点満点

```
チェック項目:
□ h1が1つだけ存在するか                  （5点）
□ h1 → h2 → h3 の順序が守られているか   （5点）
□ h2が2つ以上あるか（コンテンツの深さ）  （5点）

減点条件:
- h1が複数: -5点
- 見出しの階層スキップ（h1→h3など）: -3点
- h2が1つ以下: -3点
```

### 5. 内部リンク — 15点満点

```
チェック項目:
□ 内部リンクが2本以上あるか              （10点）
□ リンクアンカーテキストが説明的か       （5点）
  （「こちら」「ここ」は非推奨）

スコアリング:
  内部リンク 3本以上: 10点
  内部リンク 2本: 7点
  内部リンク 1本: 3点
  内部リンク 0本: 0点
```

### 6. 画像alt属性 — 10点満点

```
チェック項目:
□ 全画像にalt属性が設定されているか      （7点）
□ alt属性が空文字でないか               （3点）

減点条件:
- alt未設定の画像1枚につき: -3点
- alt属性が空文字: -2点
- altに「image」「img」など無意味な値: -1点
```

## 出力フォーマット

```markdown
## 🔍 SEOレビュー結果

**総合スコア: {score}/100** ({rating})

| 検査項目 | スコア | 満点 | 判定 |
|---------|-------|------|------|
| タイトル | {s1} | 20 | {emoji} |
| メタディスクリプション | {s2} | 20 | {emoji} |
| キーワード密度 | {s3} | 20 | {emoji} |
| 見出し構造 | {s4} | 15 | {emoji} |
| 内部リンク | {s5} | 15 | {emoji} |
| 画像alt属性 | {s6} | 10 | {emoji} |

### 🚨 Critical（必須修正）
{criticalIssues or "なし"}

### ⚠️ Warning（推奨修正）
{warningIssues or "なし"}

{suggestions}
```

**判定基準:**
- 80〜100点: ✅ 合格（publishedへの移行可）
- 60〜79点: ⚠️ 要改善
- 60点未満: ❌ 不合格（draftに戻す）

## 自動修正できる問題

以下の問題は、ユーザーの確認なしに自動修正する:
- メタディスクリプションが160文字を超えている場合 → 160文字に切り詰め
- タイトルが60文字を超えている場合 → 要約して60文字以内に
- 画像にalt属性がない場合 → 画像ファイル名から推測してalt属性を追加

## ユーザー確認が必要な問題

- 内部リンク先の提案（リンク先のURLはユーザーが判断する）
- キーワード密度が低い場合の本文修正（内容に影響するため）
